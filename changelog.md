'''
# Changelog

## v1.0.0 (2025-11-03)

### ✨ 신규 기능 (Features)

- **사용자 인증**: Supabase Google OAuth를 이용한 회원가입 및 로그인 기능 구현.
- **권한 시스템**: `admin`, `panel`, `guest` 역할에 따른 3단계 권한 관리 시스템 도입.
- **차(Tea) 관리**: 관리자(`admin`)가 새로운 차 정보를 등록, 수정, 삭제할 수 있는 기능 추가.
- **품평(Assessment) 작성**: `panel` 및 `admin` 권한 사용자가 차에 대한 품평을 작성하고 점수를 기록하는 기능 구현.
- **품평 조회**: 차 상세 페이지에서 해당 차의 모든 품평 기록을 조회하는 기능 추가.
  - `guest`는 평균 점수만 조회 가능.
  - `panel` 및 `admin`은 개별 품평 상세 내용 조회 가능.
- **데이터 시각화**: 차 상세 페이지에서 품평 점수 평균을 레이더 차트로 시각화.
- **데이터 검색**: 메인 페이지에서 차 이름을 기준으로 실시간 검색 기능 구현.
- **관리자 페이지**: 사용자 권한 관리, 전체 차 목록 조회 기능이 포함된 관리자 전용 페이지 추가.
- **데이터 내보내기**: 관리자가 전체 품평 데이터를 `CSV` 및 `JSON` 형식으로 다운로드할 수 있는 기능 구현.

### 🎨 디자인 및 UI (Design & UI)

- 제공된 디자인 샘플을 기반으로 전체 UI/UX 구현.
- **주요 페이지**: 메인(차 목록), 차 상세, 품평 작성, 관리자 페이지.
- **컴포넌트**: Header, SearchBar, TeaTable, RadarChart 등 재사용 가능한 컴포넌트 구현.
- **스타일링**: TailwindCSS를 사용한 반응형 디자인 적용.

### 🏗️ 아키텍처 및 백엔드 (Architecture & Backend)

- **프레임워크**: Next.js 14 App Router를 사용하여 서버 컴포넌트와 클라이언트 컴포넌트를 효율적으로 구성.
- **데이터베이스**: Supabase PostgreSQL을 사용하여 데이터베이스 스키마 설계 및 구축.
- **보안**: Supabase의 Row Level Security (RLS)를 적용하여 각 사용자 역할에 따른 데이터 접근 제어.
- **API**: Next.js Route Handlers를 사용하여 서버 사이드 로직 및 데이터 페칭 처리.

'''
